Cloudflare上配置域名解析

RZHANYUAN Lv1
  2025-11-21 16:51:39 2025-11-21 16:51:39 Created   2025-11-21 17:08 2025-11-21 17:08 Updated  

以下是在 Cloudflare 上配置域名解析以加速 GitHub Pages 访问的详细步骤：

一、准备工作

  1. 拥有一个已注册的域名（如在腾讯云等平台购买）。
  2. 注册并登录 Cloudflare 账号。

二、添加域名到 Cloudflare

  1. 登录 Cloudflare 后，点击 “Add a Site”，输入你的域名（如 yourdomain.com），点击 “Add Site”。
  2. Cloudflare 会自动扫描域名的 DNS 记录，等待扫描完成后点击 “Continue”。
  3. 选择免费套餐（Free），点击 “Continue”。
  4. 按照提示将域名的 Nameserver 切换为 Cloudflare 提供的服务器（需在域名注册商处操作，如阿里云）。切换后等待 DNS 生效（通常几分钟到 24 小时）。

在购入域名平台（如腾讯云）修改域名 Nameserver 为 Cloudflare 服务器的具体操作步骤：

步骤 1：获取 Cloudflare 的 Nameserver
  1. 登录 Cloudflare 控制台，进入你添加的域名管理页面。
  2. 在 “DNS” 选项卡中，找到 Cloudflare 分配给你的两个 Nameserver 地址（例如 marcus.ns.cloudflare.com 和 tina.ns.cloudflare.com），复制这两个地址。
步骤 2：登录腾讯云域名控制台
  1. 打开腾讯云官网，登录后进入域名注册控制台（路径：控制台 → 产品 → 域名与网站 → 域名注册）。
  2. 在 “我的域名” 列表中，找到你要修改的域名，点击管理。
步骤 3：修改 DNS 服务器
  1. 在域名详情页的 “基本信息” 栏，点击 “DNS 服务器” 后的修改按钮。
  2. 选择自定义 DNS，将步骤 1 中复制的 Cloudflare Nameserver 地址分别填入两个输入框。
  3. 点击提交，完成修改。
步骤 4：等待 DNS 生效
  1. 修改后，通常需要 10 分钟到 24 小时 让 DNS 全球生效。你可以在 Cloudflare 控制台点击 “立即检查名称服务器”，确认状态变为 “活跃” 即表示配置成功。

三、配置 DNS 解析记录

  1. 进入 Cloudflare 控制台，点击你的域名进入管理界面，选择 “DNS” 选项卡。

  2. 添加以下两条 DNS 记录：

(1)记录类型 A（用于裸域名，如 yourdomain.com）：
名称：@
IPv4 地址：填写 GitHub Pages 的 IP 地址（185.199.108.153、185.199.109.153、185.199.110.153、185.199.111.153，可全部添加）
代理状态：开启（橙色云朵图标）

(2)记录类型 CNAME（用于子域名，如 www.yourdomain.com）：
名称：www
目标：XIAOBAI.github.io（你的 GitHub Pages 仓库地址）
代理状态：开启（橙色云朵图标）

四、在 GitHub 中配置自定义域名

  1. 进入你的 GitHub 仓库 XIAOBAI.github.io，点击 “Settings”→“Pages”。
  2. 在 “Custom domain” 中输入你的域名（如 yourdomain.com 或 www.yourdomain.com），点击 “Save”。
  3. 仓库根目录会自动生成 CNAME 文件，确认内容为你的域名。

五、验证生效

  1. 等待 DNS 解析和 Cloudflare 配置生效（通常几分钟到 1 小时），成功之前一定会有一定的等待时间，请不要着急。
  2. 当 Cloudflar 显示你的域名为 Active 状态之后访问你的自定义域名即可正常访问博客，且国内访问速度会显著提升。
