搭建hexo博客网站

RZHANYUAN Lv1
  2025-11-20 16:20:42 2025-11-20 16:20:42 Created   2025-12-07 18:05:59 2025-12-07 18:05:59 Updated  

使用静态站点生成器 + GitHub Pages。
这个方案非常适合新手用户，它兼顾了自定义灵活性、高性能和零服务器成本。你可以把它想象成用代码搭建一个专属的、超快速的博客。
我们以目前最流行的 Hexo 为例，一步步带你从无到有搭建起来。

核心原理
1.本地写作：你在自己的电脑上使用 Markdown 格式写文章。
2.生成静态文件：通过 Hexo 工具，将你的 Markdown 文章、配置和主题，一次性编译成纯 HTML、CSS 和 JavaScript 文件。
3.部署：将这些生成的静态文件上传到 GitHub Pages。
4.访问：用户访问你的博客域名，GitHub Pages 就会直接返回这些静态文件，速度非常快。

第四步：美化博客（更换主题）

默认主题比较简约，你可以换成更美观的主题。Hexo 社区有大量免费主题。

1.挑选主题

去 Hexo 主题官网 浏览，找到一个你喜欢的主题。

2.安装主题

以 Anisina（按照例子操作可直接用） 为例，在你的博客根目录 (my-blog) 下，使用 Git 将主题克隆到 themes/Anisina 文件夹。

3.应用主题

打开博客根目录下的配置文件 _config.yml，找到 theme: 字段，将其值从 landscape 修改为 Anisina。

4.配置主题（可有可无）

每个主题都有自己的配置文件。为了方便管理，建议将主题的配置文件（themes/Anisina/_config.yml）复制一份到博客根目录，并重命名为 _config.Anisina.yml。这样你所有的个性化配置都可以放在这里，升级主题时不会丢失。

你可以根据主题的文档，修改 _config.Anisina.yml 文件来定制颜色、菜单、社交链接等。修改后，重启本地服务器即可看到效果。

对主题不满意可以去学习 Hexo 官方主题开发文档 来魔改属于你自己的主题。

第五步：部署到 GitHub Pages

当你在本地满意后，就可以把它发布到 GitHub Pages 上，让全世界看到了。

第一步：创建 GitHub 仓库

1.登录你的 GitHub 账号。
2.在页面右上角，点击 “+” 号图标，选择 “New repository”。
3.在 “Repository name” 字段中，必须填写 <你的GitHub用户名>.github.io。
例如：如果你的 GitHub 用户名是 XIAOBAI，那么仓库名就必须是 XIAOBAI.github.io。这是 GitHub Pages 识别个人主页的特殊命名规则。
4.其他选项可以保持默认，然后点击 “Create repository”。

注意：上文你的 GitHub Pages 博客在国内需翻墙才可可正常访问，可通过以下两种主流方案实现不翻墙访问：

方案一：绑定自定义域名（推荐）

通过将 GitHub Pages 关联到自己的域名，结合国内 CDN 加速，可大幅提升访问速度。

步骤 1：购买并备案域名
● 在阿里云、腾讯云等平台购买域名（如 yourname.com）。
● 若使用国内服务器或 CDN，需完成域名备案（约 1-2 周）；若仅用于 GitHub Pages 加速，可选择海外注册商（如 Namecheap）免备案。

步骤 2：配置域名解析
以 Cloudflare 为例（免费 CDN，支持全球加速）：
1.注册 Cloudflare 账号，添加你的域名。
2.在 DNS 解析中添加两条记录：
● CNAME 记录：www 指向 XIAOBAI.github.io（子域名如 www.yourname.com）。
● A 记录：若使用裸域名（如 yourname.com），需添加 GitHub Pages 的 IP 地址（185.199.108.153、185.199.109.153、185.199.110.153、185.199.111.153）。

开启 Cloudflare 的 CDN 功能（默认开启），并将域名的 Nameserver 切换为 Cloudflare 提供的服务器。

步骤 3：在 GitHub 中配置自定义域名
● 进入你的 GitHub 仓库 XIAOBAI.github.io → Settings → Pages。
● 在 Custom domain 中输入你的域名（如 www.yourname.com），点击保存。

仓库根目录下会自动生成 CNAME 文件，确认内容为你的域名。

步骤 4：验证与生效
● 等待 DNS 解析生效（通常几分钟到 24 小时），访问你的自定义域名即可看到博客，且国内访问速度会显著提升。

方案二：迁移到国内代码托管平台（备选）

若对 GitHub 依赖较低，可将博客迁移到国内平台，实现完全无墙访问。

步骤 1：导入仓库到 Gitee
● 注册 Gitee 账号，点击「+」→「从 GitHub/GitLab 导入仓库」，输入你的 GitHub 仓库地址（https://github.com/XIAOBAI/XIAOBAI.github.io.git）。
● 导入完成后，进入仓库 →「服务」→「Gitee Pages」，选择部署分支（如 main），点击「启动」。

步骤 2：访问国内域名
● 部署成功后，可通过 Gitee 提供的域名（如 yourname.gitee.io）访问博客，国内访问速度极快。
● 若需自定义域名，流程与 GitHub Pages 类似，需在 Gitee 中配置并解析域名。

总结:
● 优先选择方案一：绑定自定义域名 + Cloudflare CDN，既保留 GitHub 的生态，又能实现国内高速访问。
● 方案二适合轻量需求：迁移到 Gitee 可完全规避翻墙问题，但功能和生态不如 GitHub 完善。

两种方案均可实现博客 “无墙” 访问，可根据你的技术熟悉度和需求选择。
